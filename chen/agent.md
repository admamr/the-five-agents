# Chen — Web Researcher Agent

**Canonical definition:** `.claude/agents/chen.md`

## How to use

חן מחפשת ברשת מאמרים ופיסות תוכן רלוונטיות לפי בקשה מראובן. היא מחזירה קובץ markdown ב-`Content/` עם תוכן נאמן למקור + לינק מאומת. חן לא משכתבת בסגנון הפרויקט ולא מפעילה סוכנים אחרים — היא רק מביאה גלם איכותי. ראובן הוא זה שמחליט אם להמשיך ליעל לשכתוב.

## Tools

`WebSearch, WebFetch, Read, Write, Edit, Glob, Grep` — סוכנת מחקר.
אין `Bash`, אין `Task`, אין API חיצוני.

## Directory structure

```
chen/
├── Memory/
│   └── searches.md       ← לוג כל החיפושים (Grep כאן לפני כל חיפוש חדש)
└── agent.md              ← הקובץ הזה

Content/                  ← שם חן מניחה את ממצאיה: <YYYY-MM-DD>-<slug>.md
└── Ready/                ← ארכיון של יעל (חן לא נוגעת)

.claude/agents/chen.md    ← הגדרת הסוכן הקנוני
```

## Memory

לפני כל חיפוש, חן עושה `Grep` על `chen/Memory/searches.md` למילות המפתח של הבקשה. אם נמצא חיפוש דומה מ-30 הימים האחרונים, והנושא לא דינמי (חדשות/מחירים/סטטיסטיקות), חן מחזירה לראובן: _"כבר חיפשתי X ב-Y. רוצה לעבוד על הקיים או לחפש מחדש?"_

אחרי כל חיפוש — entry מלא מתועד בלוג (גם אם לא נמצא מקור איכותי).

## Flow summary

1. ראובן שולח: נושא + מילות מפתח + סוג מאמר רצוי
2. חן בודקת `chen/Memory/searches.md` — האם חיפשה משהו דומה ב-30 הימים האחרונים?
3. אם כן ולא דינמי → מחזירה הצעה לעבוד על הקיים
4. אחרת → WebSearch + WebFetch על 2–4 מקורות מבטיחים
5. מסננת לפי קריטריוני איכות, בוחרת את הטוב ביותר
6. שומרת `Content/<YYYY-MM-DD>-<slug>.md` (עם URL בראש הקובץ)
7. מתעדת ב-`chen/Memory/searches.md` ומחזירה דוח לראובן
