# Obsidian Vault Workflow

## Overview

הפרוטוקול המחייב לניהול ה-vault. **חובה** לבצע לפני כל משימה (Phase 1: קריאת קובץ הנושא + recency context) ואחרי כל משימה (Phase 2: עדכון/יצירת קובץ נושא עם Session Log). מבנה הוולט: תיקייה אחת לכל סוג תוכן, קובץ אחד לכל נושא.

### מבנה תיקיות

| תיקייה | שימוש |
|---|---|
| `vault/Meeting Notes/` | קוד, ארכיטקטורה, החלטות |
| `vault/Content Briefs/` | בריפים עריכתיים |
| `vault/Publishing Log/` | ריצות פרסום |
| `vault/Brand Guidelines/` | קול, ויזואלים, טון |
| `vault/Skills/` | תיעוד skills (הוסף ידנית לפרויקט זה) |
| `vault/Project Files/` | תיעוד קבצי פרויקט (הוסף ידנית) |

### פורמט קובץ נושא

```
# <כותרת>
## Overview
## Open Questions
## Session Log
### YYYY-MM-DD — <כותרת> [status]
```

### Status tags

`[shipped]` | `[spiked]` | `[wip]` | `[reverted]` | `[planned]` | `[debug]`
